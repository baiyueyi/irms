# Rule 06｜API 规则

- 所有接口统一使用 `/api/...`。
- 禁止使用 `/api/v1/...` 或 `/api/v2/...`。
- API 设计优先清晰、稳定、可联调。
- 创建/更新接口优先使用明确 request DTO；列表/详情返回优先使用 response DTO 或 VO。
- 请求与响应结构保持前后一致，错误响应形状统一。
- 分页接口统一支持页码、页大小、关键字、类型等最小筛选参数。
- 写接口严格校验；关键 query 参数非法值明确返回 400，不要静默忽略。
- 未经明确确认，不擅自改变已定稿接口路径、权限逻辑、字段名称。
- CLI 初始化属于命令行流程，不写成 HTTP 接口。
